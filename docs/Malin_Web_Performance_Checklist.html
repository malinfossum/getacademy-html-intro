Web Performance Checklist — Dark Elegance Edition

Quick, practical steps to keep your portfolio fast, accessible, and elegant on every device.

Images (Hero Banner & Media)

  • Prefer WebP for modern browsers, with JPG/PNG fallback via <picture>.
  • Export sizes: desktop 1920×960, mobile 1000×1000 (when available).
  • Compression: WebP ~80% quality; JPG ~75–85%; strip metadata for web.
  • Add width/height (or CSS aspect-ratio) to prevent layout shift.
  • Preload critical hero image with <link rel="preload" as="image">.
  • Use sizes="100vw" so the browser picks the right resource.
  • Set decoding="async" and loading="lazy" for below-the-fold media.

CSS & Rendering

  • Respect prefers-reduced-motion; disable heavy animations for those users.
  • Keep animations to opacity/transform for GPU-friendly transitions.
  • Use mask-image or gradient edges for soft blends without large PNGs.
  • Minimize unused CSS; ship only what the page needs.

Fonts (Urbanist & Outfit)

  • Preconnect to fonts.gstatic.com and use display=swap to avoid FOIT.
  • Only load the weights/styles you use (Urbanist 400/700; Outfit 400/600/400-italic).
  • Set fallbacks: system-ui, -apple-system, Segoe UI, Roboto, Arial, sans-serif.

HTML Markup

  • Use <picture> with type hints (image/webp, image/jpeg) + media queries for responsive sources.
  • Include descriptive alt text for accessibility and SEO.
  • Place hero near top of DOM; add fetchpriority="high" to the primary <img>.

Caching & Hosting

  • Version image filenames when you update them (e.g., _v2).
  • Use GitHub Pages (or a CDN) which serves static assets efficiently.
  • Keep your repo tidy: /assets/images/ for media; document sizes in README.

Quick Audit (before publishing)

  • Lighthouse: check Performance & Accessibility scores.
  • Largest Contentful Paint (LCP) < ~2.5s on slow 4G emulation.
  • No layout shift when the banner loads (CLP stable).
  • Dark mode contrast: text vs background ≥ 4.5:1 where applicable.
